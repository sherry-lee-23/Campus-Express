
# 菜鸟驿站配送调度系统 —— 项目框架分析

## 一、项目概述

这是一个**基于贪心策略的快递配送调度模拟系统**，用于解决菜鸟驿站校园配送场景下的路径规划与任务调度问题。系统从文本文件读入地图、包裹清单和小车参数，提供命令行交互界面，支持最短路查询、不满意度最小化、带容量成本计算、退货回收（TSP）、双车协同以及策略对比等六项核心任务。

### 核心任务一览

| 任务         | 描述                              | 算法类型                    |
| ------------ | --------------------------------- | --------------------------- |
| **T1** | 查询任意两点间最短距离与路径      | Dijkstra（手写堆）          |
| **T2** | 最小化不满意度之和 Σ(Di - Si)    | 贪心（先到先装 + 最近邻）   |
| **T3** | 带容量的运送成本计算 + 超时统计   | 贪心（按距离分组 + 最近邻） |
| **T4** | 退货回收（容量无限，TSP）         | 精确 DP / 启发式 2-opt      |
| **T5** | 双车协同配送                      | 贪心分配 + T3 调度          |
| **T6** | 多策略对比（FIFO/EDF/综合评分等） | 策略模式对比                |

## 三、核心数据结构

### 3.1 数据模型（Delivery.h）

| 结构体                       | 字段                                                                              | 说明            |
| ---------------------------- | --------------------------------------------------------------------------------- | --------------- |
| **Package**            | `id`, `weight`, `dest`, `arrive`, `deadline`                            | 包裹信息        |
| **Car**                | `speed`, `carWeight`, `capacity`                                            | 小车参数        |
| **ShortestPathResult** | `distance`, `path`, `reachable`                                             | T1 查询结果     |
| **Trip**               | `packageIndices`, `route`, `departureTime`, `returnTime`, `totalWeight` | 单趟配送方案    |
| **T2Result**           | `trips`, `totalDissatisfaction`                                               | T2 结果         |
| **SegmentCost**        | `from`, `to`, `distance`, `cargoWeight`, `segmentCost`                  | T3 单段成本     |
| **T3Result**           | `trips`, `allSegments`, `totalCost`, `timeoutCount`                       | T3 结果         |
| **T4Result**           | `route`, `totalTime`                                                          | T4 退货回收结果 |
| **T5Result**           | `car1Trips`, `car2Trips`, `totalCost`, `timeoutCount`                     | T5 双车结果     |

**全局常量：**

- `INF = 1e18`：表示不可达
- `EPS = 1e-9`：浮点数比较容差

### 3.2 图模型（LGraph.h）

| 结构体                 | 字段                                 | 说明                           |
| ---------------------- | ------------------------------------ | ------------------------------ |
| **LocationInfo** | `id`, `x`, `y`, `is_station` | 节点信息（坐标仅供可视化）     |
| **EdgeNode**     | `from`, `to`, `weight`         | 有向边（构造时需显式添加双向） |

**LGraph 类方法：**

| 方法                     | 返回值                        | 说明                      |
| ------------------------ | ----------------------------- | ------------------------- |
| `VertexCount()`        | `int`                       | 获取顶点数                |
| `EdgesCount()`         | `int`                       | 获取边数                  |
| `exist_vertex(id)`     | `bool`                      | 检查顶点是否存在          |
| `exist_edge(from, to)` | `bool`                      | 检查边是否存在            |
| `AllPlaces()`          | `vector<int>`               | 获取所有顶点 ID（已排序） |
| `AllEdges()`           | `vector<EdgeNode>`          | 获取所有边                |
| `GetVertex(id)`        | `const LocationInfo&`       | 获取顶点信息（抛异常）    |
| `GetNeighbors(id)`     | `const map<int, EdgeNode>&` | 获取邻接表（抛异常）      |

**重要：** LGraph 存储的是**有向边**。构建无向图时，调用方需在 edges 中同时添加 `(u,v,w)` 和 `(v,u,w)`。

### 3.3 手写数据结构

#### MinHeap（Delivery::MinHeap<KeyType, ValueType>）

| 方法                     | 说明     | 复杂度   |
| ------------------------ | -------- | -------- |
| `push(key, value)`     | 插入元素 | O(log n) |
| `top()`                | 查看堆顶 | O(1)     |
| `pop()`                | 弹出堆顶 | O(log n) |
| `empty()` / `size()` | 状态查询 | O(1)     |

**用途：** Dijkstra 算法的优先队列。

#### CircularQueue（Delivery::CircularQueue<T></t>）

| 方法                                  | 说明     | 复杂度    |
| ------------------------------------- | -------- | --------- |
| `push(item)`                        | 入队     | 均摊 O(1) |
| `front()`                           | 查看队首 | O(1)      |
| `pop()`                             | 出队     | O(1)      |
| `empty()` / `full()` / `size()` | 状态查询 | O(1)      |

**特点：** 底层 vector 环形数组，满时自动 2 倍扩容。

**用途：** T2 维护“已到站、等待装载”的包裹队列（FIFO）。

## 十、设计特点与架构评价

### 优点

- **分层清晰**：表示层、算法层、数据层、基础设施层职责分明
- **纯函数风格算法层**：无状态、易测试、接口直接
- **手写核心数据结构**：MinHeap、CircularQueue 加深底层理解
- **依赖注入**：CommandProcessor 通过构造函数注入所有依赖

### 待改进

- `CommandProcessor.h` 与当前算法层接口不匹配（引用了已移除的类）
- `algorithm.h` 缺少 `AllPairsResult` 定义
- T4/T5/T6 的策略参数（如 T2Strategy、T3Strategy）建议用 `enum class` 替代 `string`
- 缺少错误处理和输入校验（在 TODO 实现时需补充）
